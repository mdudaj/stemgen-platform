# TIE Source Snapshot Plan

## Purpose

Define how official TIE curriculum sources are captured reproducibly before extraction. Slice 1 implements source registration, snapshot manifests, checksums, and dashboard artifact inspection only.

## Snapshot Steps

1. Load `fixtures/curriculum/tie_sources.json` into registered `CurriculumSource` rows.
2. Fetch each Tier 1 URL with an explicit operator command, unless `--no-download` is passed.
3. Store files under `artifacts/curriculum-snapshots/<snapshot_id>/downloaded_sources/`.
4. Record URL, timestamp, HTTP status, content type, byte count, and local path in `fetch_manifest.json`.
5. Compute SHA-256 checksums for each downloaded file.
6. Record checksum values in `checksums.sha256` and `fetch_manifest.json`.
7. Freeze the snapshot UUID for downstream extraction.

## Commands

Dry run, with no artifact or database writes:

```bash
python manage.py curriculum_snapshot create --dry-run --json
```

Offline no-download snapshot:

```bash
python manage.py curriculum_snapshot create --snapshot-id local-no-download --no-download --validate --json
```

Real snapshot when network access is intentionally allowed:

```bash
python manage.py curriculum_snapshot create --validate --json
```

## Snapshot UUID

Snapshot IDs are UUIDs generated by the system when capture starts. Operators should not type or normally inspect snapshot IDs in the web UI; the snapshot description is the human-facing label. See ADR 0012.

## Failure Handling

- If a URL is unavailable, record the failure in `fetch_manifest.json`.
- Do not substitute unofficial copies without manual registry review.
- Do not overwrite an existing snapshot; create a new snapshot UUID.

## Manual Review

A human reviewer should confirm source titles, years, subject coverage, standards covered, and licensing caution before the first accepted topic shortlist.

## Dashboard Review

Staff users can inspect snapshots at `/curriculum/snapshots/`. The dashboard shows registered source count, latest snapshot status, validation status, downloaded count, failed count, and warnings count.

The snapshot detail page shows manifest paths, validation errors, warnings, captured source metadata, stored filenames, sizes, SHA-256 values, and failure messages. The page is read-only and uses review language such as "Inspect", "Review captured artifacts", "Validation status", and "Ready for extraction".

## Slice 1 Non-Goals

Slice 1 does not perform PDF text extraction, table extraction, normalized curriculum item generation, candidate topic generation, topic screening, automated LLM review, human topic selection, Viewflow workflow review, generation runs, expert evaluation, WhatsApp messaging, export bundles, replay, or deployment.
